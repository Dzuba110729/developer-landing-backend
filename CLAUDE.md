## Технологический стек

- **PHP 8.2+, Laravel 11** (Composer)
- **Хранение данных — файловое** (JSON/текстовые файлы): логи запросов, статистика, rate-limit. БД не используется — задание явно не требует, а файлового хранилища достаточно, чтобы не усложнять деплой.
- **Почта** — `Mail` фасад Laravel, локально через Mailpit/Mailtrap, в проде — любой SMTP.
- **AI-интеграция** — Anthropic API (Claude) как основной провайдер, с graceful fallback на случай недоступности (сервис должен продолжать работать без AI).
- **Документация API** — L5-Swagger (OpenAPI аннотации) либо статический `openapi.yaml` + Swagger UI.
- **Деплой** — Railway/Render (бесплатный тир) либо локальный запуск + ngrok, если деплой не получится.

## Архитектура

Слоистая структура согласно требованиям задания:

```
Controller → Service → Repository/Handler
```

- `app/Http/Controllers/Api/` — тонкие контроллеры, только приём запроса и формирование ответа
- `app/Http/Requests/` — Form Request классы для валидации (имя, телефон, email, комментарий)
- `app/Services/` — бизнес-логика: `ContactService`, `AiService` (анализ тональности/классификация/автоответ), `MailService`
- `app/Repositories/` или `app/Storage/` — работа с файловым хранилищем (логи, статистика, rate-limit)
- `app/Exceptions/Handler.php` — глобальная обработка ошибок с правильными HTTP-статусами
- `app/Http/Middleware/` — кастомный middleware логирования запросов в файл + rate limiting (throttle, file-driver кеш)

## Обязательный функционал

1. `POST /api/contact` — приём заявки, валидация, AI-обработка комментария, отправка письма владельцу + копии пользователю
2. `GET /api/health` — статус сервиса
3. `GET /api/metrics` — статистика обращений (из файла)
4. Rate limiting на `/api/contact`
5. Логирование всех запросов в файл
6. AI-функция с graceful fallback (если Anthropic API недоступен — заявка всё равно сохраняется и письмо уходит)
7. CORS настроен
8. Swagger/OpenAPI документация

## Что важно не упустить (из требований заказчика)

- "Backend без API и обработки ошибок — не считается выполненным"
- "API без AI-интеграции — тоже не считается выполненным"
- Полный цикл должен быть виден: запрос → валидация → бизнес-логика → AI → отправка → ответ
- README должен содержать: запуск, стек, архитектуру, описание эндпоинтов, AI-интеграцию (включая использованные промпты), раздел "что сделано с помощью AI", описание хранения данных
- Postman-коллекция или curl-примеры — обязательны

## Стиль кода

- PSR-12, строгая типизация (`declare(strict_types=1)` где уместно)
- Form Request вместо валидации в контроллере
- Без избыточных абстракций — задание тестовое, важна читаемость и соответствие пройденным критериям, а не "энтерпрайз"-наслоения
